# План внедрения системы 9-Slice UI

## 1. Создание вспомогательного класса NineSlice
Класс будет отвечать за отрисовку текстур с сохранением пропорций углов при масштабировании.

**Способы решения:**
- Создание класса, наследующегося от `sf::Drawable` и `sf::Transformable`.
- **Плюсы:** Легкая интеграция в существующую систему отрисовки SFML, возможность трансформации (позиция, масштаб, поворот).
- **Минусы:** Требуется ручное управление геометрией через `sf::VertexArray`.
- **Выбранный вариант:** Реализация через `sf::VertexArray` с 54 вершинами (9 секторов по 2 треугольника), так как это наиболее производительный способ в SFML для данной задачи.

**Этапы:**
- Создание `includes/ui/NineSlice.hpp` с соблюдением разделителей и стиля комментариев.
- Создание `src/ui/NineSlice.cpp` с реализацией расчета вершин.

## 2. Интеграция в класс Button
Добавление возможности использования 9-slice текстур для фонов кнопок в разных состояниях.

**Способы решения:**
- Расширение класса `Button` новыми полями для текстур и объекта `NineSlice`.
- **Этапы:**
    - Модификация `includes/ui/Button.hpp`: добавление методов `setBackgroundTextures`.
    - Модификация `src/ui/Button.cpp`: обновление методов `render`, `setPosition`, `setSize`, `setEnabled` и логики обработки событий для смены текстур.

## 3. Интеграция в класс Container
Добавление поддержки 9-slice фона для панелей и контейнеров.

**Способы решения:**
- Добавление объекта `NineSlice` и метода `setBackgroundTexture`.
